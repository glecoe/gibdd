# Система оперативного уведомления ГИБДД

Перенос Telegram-бота в полноценную систему: сервер + два мобильных приложения.

## Состав

| Компонент          | Технологии                      | Назначение                                    |
|--------------------|----------------------------------|-----------------------------------------------|
| `gibdd_backend`    | FastAPI + SQLite + JWT           | Сервер: API, БД, авторизация, рассылка пушей   |
| `eyewitness_app`   | Kotlin + Jetpack Compose         | Приложение очевидца: отправка инцидентов       |
| `officer_app`      | Kotlin + Jetpack Compose + FCM   | Приложение сотрудника: патруль, обработка, админка |

## Роли

- **Очевидец** — без логина (вход по device_id). Шлёт фото/видео/гео о нарушении.
- **Инспектор** — логин/пароль. Патрулирование, приём и закрытие инцидентов.
- **Администратор** — назначает роли (инспектор/очевидец), создаёт инспекторов, глушит свои уведомления.
- **Начальник** — всё что админ + назначает/удаляет администраторов, отключает пользователей.

## Порядок запуска

### 1. Сервер
```bash
cd gibdd_backend
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```
При первом запуске создаётся начальник `+70000000000` / `admin123`.
Swagger: http://localhost:8000/docs

### 2. Приложение очевидца
Открыть `eyewitness_app` в Android Studio, запустить. Работает сразу.

### 3. Приложение сотрудника
Открыть `officer_app` в Android Studio, запустить.
Войти под начальником, создать инспекторов через «Управление».

## Как проверить весь сценарий

1. Запусти сервер.
2. В приложении сотрудника войди начальником → создай инспектора.
3. Выйди, войди инспектором → нажми «Начать патруль».
4. В приложении очевидца опиши нарушение, приложи фото, отправь.
5. У инспектора в ленте появляется инцидент (через опрос, либо пуш если настроен Firebase) → «Принять» → «Закрыть».

## Доставка уведомлений

Сейчас работает двумя путями:
- **Опрос сервера** — приложение сотрудника на патруле опрашивает сервер каждые 15 секунд. Работает всегда, без настройки.
- **Push через Firebase** — мгновенные уведомления. Требует настройки (см. `officer_app/README.md`).

## Важно про адрес сервера

- Эмулятор Android видит хост-машину как `10.0.2.2` (адрес по умолчанию).
- Реальный телефон: укажи IP компьютера в Wi-Fi сети через экран настроек в приложении.
- Сервер должен быть запущен с `--host 0.0.0.0`, телефон и ПК — в одной сети.
