# Приложение очевидца — ГИБДД

Android-приложение для граждан. Позволяет анонимно сообщить о пьяном
водителе / нарушении: приложить описание, фото/видео, геолокацию.
Сообщение уходит на сервер FastAPI и рассылается инспекторам на патруле.

## Функции

- Вход без регистрации (по device_id устройства)
- Описание нарушения (марка, цвет, номер, что происходит)
- Прикрепление нескольких фото из галереи или съёмка камерой
- Определение геолокации одной кнопкой
- Отправка на сервер
- Просмотр истории своих сообщений со статусами (новый / принят / обработан)
- Настройка адреса сервера прямо в приложении (без пересборки)

## Сборка

1. Открой папку `eyewitness_app` в **Android Studio** (Ladybug / Koala или новее).
2. Дождись синхронизации Gradle (нужен интернет для скачивания зависимостей).
3. Подключи телефон или запусти эмулятор.
4. Run → Run 'app'.

## Настройка адреса сервера

По умолчанию приложение стучится на `http://10.0.2.2:8000/` —
это адрес хост-машины из эмулятора Android.

Если тестируешь на реальном телефоне:
1. Запусти сервер: `uvicorn app.main:app --host 0.0.0.0 --port 8000`
2. Узнай IP компьютера в Wi-Fi сети (например `192.168.1.50`)
3. В приложении: Настройки (шестерёнка) → введи `http://192.168.1.50:8000` → Сохранить
4. Телефон и компьютер должны быть в одной Wi-Fi сети

## Стек

- Kotlin 2.0 + Jetpack Compose + Material3
- Retrofit 2 + OkHttp (сеть)
- DataStore Preferences (хранение device_id, токена, URL)
- Coil (отображение фото)
- FusedLocationProviderClient (геолокация)
- Navigation Compose (навигация между экранами)

## Структура

```
app/src/main/java/com/gibdd/eyewitness/
├── MainActivity.kt              — точка входа, навигация
├── data/
│   ├── AppPreferences.kt        — DataStore: device_id, токен, URL
│   ├── IncidentRepository.kt    — бизнес-логика: вход, отправка, история
│   └── LocationHelper.kt        — получение координат
├── network/
│   ├── ApiFactory.kt            — создание Retrofit с токеном
│   ├── ApiService.kt            — описание API-эндпоинтов
│   └── Dto.kt                   — модели данных (DTO)
└── ui/
    ├── MainViewModel.kt         — состояние экранов
    ├── theme/Theme.kt           — цветовая тема
    └── screens/
        ├── ReportScreen.kt      — отправка инцидента
        ├── HistoryScreen.kt     — история сообщений
        └── SettingsScreen.kt    — настройка URL сервера
```
